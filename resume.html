About Me

저는 비즈니스 문제를 기술로 해결하는 과정에 매력을 느끼는 프론트엔드 개발자입니다. React와 TypeScript를 주력으로 사용하며, 성능과 접근성을 고려한 컴포넌트 설계에 집중합니다.

특히 깃허브 데이터를 활용해 자동으로 포트폴리오를 생성하고, 사용자가 빠르게 자신의 작업을 보여줄 수 있는 경험을 만드는 것을 목표로 이 프로젝트를 진행했습니다.

이 프로젝트에서는 데이터 수집(깃허브), 템플릿 기반 렌더링, 간단한 AI 요약 기능을 통합하여 사용자가 최소한의 설정으로 완성도 높은 포트폴리오를 만들 수 있도록 했습니다.

문제 정의

많은 개발자가 깃허브에 프로젝트를 올리지만 이를 보기 좋고 설명하기 쉬운 포트폴리오로 바꾸는 데 시간이 걸립니다. 수작업으로 레이아웃을 만들고 내용을 정리하는 대신 자동으로 깃허브 데이터를 수집·가공해 빠르게 포트폴리오를 생성하고자 했습니다.

아키텍처 / 설계

Next.js(App Router) 기반 SPA/SSR 혼합 아키텍처를 사용했습니다. 사용자 인증은 GitHub OAuth로 처리하고, 서버 라우트에서 토큰을 이용해 GitHub API로 데이터를 수집합니다. 데이터는 캐시(Map/옵션 Redis)로 보관해 호출 부하를 줄이고, 템플릿은 React 컴포넌트(동적 import)로 분리해 클라이언트에서 필요할 때만 로드합니다.

핵심 기능

깃허브 자동 수집 및 증강

OAuth로 접근 토큰을 받아 사용자의 repos, 언어 통계, 기여도, 커밋 수 등을 수집합니다. 각 리포지토리에 대해 병렬 제한(mapWithLimit)과 캐시를 적용해 안정적으로 데이터를 가져오며, 언어·기여자 수 등 추가 정보를 보강해 템플릿에 적합한 형식으로 제공합니다.

템플릿 기반 동적 렌더링

여러 템플릿을 React 컴포넌트로 분리하고 dynamic import로 필요 시 로드합니다. Tailwind 기반 공통 유틸 컴포넌트를 활용해 템플릿 간 스타일 중복을 줄이고, 선택된 프로젝트/프로필 데이터를 주입해 즉시 미리보기를 제공합니다.

  • 핵심 API 응답 지연 최소화(캐시 적용으로 반복 호출 성능 향상)

회고 (Retrospective)

배운 점

GitHub API의 한계(레이트 리밋, 검색 API 특성)를 다루며 백오프/재시도 전략과 캐시의 중요성을 체감했습니다. 또한 dynamic import와 Tailwind 조합으로 템플릿 관리가 용이하다는 것을 확인했습니다.

개선점

대량의 템플릿(수천 이상)을 지원하려면 현재 구조를 manifest/외부 번들 방식으로 전환해야 합니다. 또한 서버 로그 마스킹 및 민감정보 처리(토큰 관리) 부분을 강화하고, 자동 테스트(단위·통합)를 추가해 안정성을 높이고자 합니다.
