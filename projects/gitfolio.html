Problem

개발자로서 코드는 매일 업데이트되지만, 포트폴리오는 멈춰있는 경우가 많았습니다. GitHub의 기록을 다시 포트폴리오로 정리하는 과정은 번거롭고 반복적인 작업이었습니다.

Solution

복잡한 과정 없이 GitHub 연동 버튼 하나만 누르면 내 코드가 근사한 포트폴리오 사이트로 즉시 변환되는 경험을 설계했습니다.

Frontend (View)

화면과 서버 로직을 명확히 분리하고, 템플릿 확장을 위해 "데이터(Model)"와 "디자인(View)"을 분리하는 전략을 취했습니다. 동일한 데이터 모델을 여러 템플릿 컴포넌트에 주입하는 구조로 유연성을 확보했습니다.

Backend (Logic)

GitHub OAuth로 인증 처리 후, GitHub API로 원천 데이터를 확보합니다. AI 분석 파이프라인을 거쳐 풍부한 설명 텍스트로 변환된 데이터는 Firebase에 저장되어 빠른 로딩을 보장합니다.

주요 기능 소개

원클릭 데이터 동기화

사용자의 GitHub 계정을 분석하여 소유 또는 기여한 모든 저장소(Repository)를 불러옵니다.

AI 기반 회고/요약

단순한 데이터 나열을 넘어, 생성형 AI가 각 프로젝트의 README.md 파일과 커밋 로그(Commit History)를 심층 분석합니다. 이를 바탕으로 '프로젝트 개요', '핵심 기능', '나의 기여도'를 문장 형태로 자동 작성해 줍니다. "무엇을 써야 할지" 막막한 초안 작성 단계를 AI가 해결해 주어, 사용자는 검토와 수정에만 집중할 수 있습니다.

템플릿 즉시 교체

확보된 데이터와 디자인(View)을 완벽하게 분리했습니다. '모던(Modern)', '클래식(Classic)', '미니멀(Minimal)' 등 다양한 템플릿을 데이터 손실 없이 즉시 교체할 수 있습니다.
실시간 미리보기(Preview)를 통해 내 프로젝트 성격에 가장 잘 어울리는 디자인을 직관적으로 선택하세요.
